# Iteration V2 说明

## 1. 本次可继续优化的点（超出知识库本身）
- 安全性：新增登录失败次数限制、密码复杂度策略、账号锁定时长配置。
- 可观测性：对 `qa/ingestion` 接口增加耗时分位数统计（P95/P99）与告警阈值。
- 稳定性：文档解析和向量化改为异步任务队列，避免大文件阻塞请求线程。
- 成本优化：为历史冷文档增加低频索引策略，降低 ES 存储与检索成本。
- 体验优化：管理员操作（审核/批量删除/改角色）增加二次确认与审计原因必填。

## 2. 新增测试账号（启动后自动创建）
> 由 `DataBootstrapRunner` 启动时写入，若账号已存在则跳过，不覆盖原数据。

- 普通用户：`user_qa` / `Qa@123456`
- 文档管理员：`doc_admin` / `Doc@123456`
- 超级管理员：`super_admin` / `Super@123456`

## 3. 数据库迁移说明
- 全量重建：执行 `docs/sql/schema.sql`
- 增量迁移：执行 `docs/sql/migration_v2.sql`

### 推荐顺序（旧库升级）
1. 备份数据库 `rag_knowledgehub`
2. 执行 `migration_v2.sql`（只加字段/表，不删旧数据）
3. 重启后端服务
4. 首次启动后系统会自动补默认账号和基础配置

## 4. 关键变更点
- RBAC 三级角色：`USER / DOC_ADMIN / SUPER_ADMIN`
- 文档能力：多格式 + 批量上传 + 审核 + 标签 + 版本 + 敏感提示
- 问答能力：混合检索 + 过滤 + 5轮上下文 + 摘要 + 溯源跳转
- 系统能力：操作日志、密码修改、账号管理、系统配置
